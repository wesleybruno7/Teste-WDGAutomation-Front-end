Pré-requisitos

A solução deve ser publicada no github e o link do repositório enviado para a avaliação.
No repositório deve existir um arquivo readme com informações de como executar a aplicação.

Não utilizar templates prontos.
Pode utilizar biblioteca de UI, exemplos: materialize, angular-ui, bootstrap, material, etc.

O que será avaliado

  • Completude dos requisitos
  • Design da arquitetura
  • Design do código
  • User Experience

Single Page App

Criar um Single Page Application com a tecnologia que você conhece (Angular, React, CSS, LESS, SASS, etc).

A aplicação deve contemplar:

  • Autenticação
  • Listagem de usuários
  • Exclusão de um usuário
  • Atualização de um usuário
  • Página 404

API

A API que deverá ser usada para obter e atualizar os dados é https://reqres.in/api. Essa é uma API fake, os dados não serão realmente salvos ou excluídos.
Requisito: em qualquer rota API mencionada nesse documento, usar o parâmetro ?delay=2 para simular latência na chamada.
Leia a documentação de como funciona essa API em https://reqres.in/.

Login

Mostrar a página de login ao acessar http://<domain>/#/login ou a rota padrão http://<domain>/#/.
Usar a API POST https://reqres.in/api/login

{
    "email": "peter@klaven",
    "password": "cityslicka"
}

Se a API retornar status 200, navegar para a página de listagem de usuários http://<domain>/#/users. Se a API retornar outro status, mostrar erro e não navegar.

Importante: utilizar o bearer token retornado no Header Authorization nas próximas chamadas da API - apesar da API não validar o usuário logado, é requisito fazer isso no código.

Listagem de Usuários

Mostrar a página de usuários ao acessar http://<domain>/#/users.
Usar a API GET https://reqres.in/api/users. Essa API retorna os dados paginados, portanto use o parâmetro ?page=<NUMERO_PAGINA> para mostrar os dados de outras páginas.

A rota mostrada no navegador deve ser compatível com os dados mostrados de cada página de users.
Por exemplo, ao acessar http://<domain>/#/users?page=2, deve-se mostrar os usuários da página 2.

A página não deve usar Tabela para listar os usuários
Deve existir um link ou botão em cada usuário listado para ir para a página de edição desse usuário: http://<domain>/#/users/<ID> .

Exclusão de Usuário

Na página de listagem, deve existir um botão para excluir o usuário.
Ao clicar no botão, deve-se confirmar a ação, e caso afirmativa, excluir o usuário através da API DELETE https://reqres.in/api/users/<ID>.

Edição de Usuário

Mostrar a página de edição de usuário ao acessar http://<domain>/#/users/<ID>.
Usar a API GET https://reqres.in/api/users/<ID> para obter os dados do usuário especifico.

A página deve permitir editar os campos first_name e last_name.
Exemplo de chamada para atualizar: PUT https://reqres.in/api/users/<ID>

{
    "first_name": "morpheus",
    "last_name": "zion resident"
}

404

Se o usuário acessar qualquer rota não mapeada, mostrar uma página de 404.
Exemplo de rotas 404: http://<domain>/#/cars, http://<domain>/#/admin, http://<domain>/#/users/9999999, etc.
